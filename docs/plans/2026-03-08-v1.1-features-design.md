# Prayer Room v1.1 Features Design

## Goal

Add 5 major features to Prayer Room: custom intervals, full offline Bible with thematic verse index, alarm display templates (including incoming-call style), i18n (FR/EN), and bundled alarm sounds.

## Architecture Summary

- Bible data: JSON files per book, lazy-loaded, sourced from getBible v2 (public domain)
- Thematic index: verse-tags.json mapping references to categories (50-200+ verses per category)
- i18n: simple key-value JSON files + locale detection + manual override
- Alarm templates: 4 swappable full-screen layouts chosen per alarm in settings
- Custom intervals: existing pill presets + "Autre" pill opening a picker modal

## Tech Stack

- React Native + Expo SDK 55, Expo Router
- AsyncStorage for settings/preferences
- expo-notifications for reliable recurring alarms
- expo-av for audio playback (dev build)
- expo-localization for locale detection
- Animated API + expo-linear-gradient for alarm template animations

## Feature 1: Custom Interval Picker

### Current State
IntervalPicker has 6 hardcoded pills: 30min, 40min, 1h, 1h30, 2h, 3h.

### Design
- Keep existing pills, add a 7th pill labeled "Autre" at the end
- Tapping "Autre" opens a bottom sheet modal with two iOS-style scroll pickers (hours: 0-12, minutes: 0-55 by 5min steps)
- User confirms with a "Valider" button
- If a custom interval is active (not matching any preset), the "Autre" pill shows gold with the value (e.g. "45 min")
- Minimum interval: 5 minutes. Maximum: 12 hours.

### Files Affected
- Modify: `src/components/IntervalPicker.tsx` (add "Autre" pill + modal)
- No type changes needed (`intervalMinutes: number` already flexible)

## Feature 2: Full Offline Bible + Thematic Verse Index

### Current State
5 hardcoded packs with 20 verses each in `src/data/builtin-packs.ts`. Categories: healing, encouragement, prosperity, faith, protection.

### Bible Data

**Translations (all public domain):**

| Key | Name | Language | Bundled |
|-----|------|----------|---------|
| ls1910 | Louis Segond 1910 | FR | Yes (default FR) |
| darby_fr | Darby | FR | Download |
| martin | Martin 1744 | FR | Download |
| kjv | King James Version | EN | Yes (default EN) |

**Storage structure:**
```

  kjv/
```

- Bundled translations (~4-5 MB each) ship with the app
- Downloadable translations stored in app filesystem via expo-file-system
- Each book is a separate JSON file, lazy-loaded when navigated to

**Bible metadata:**
```
src/data/bible-meta.json
{
  "books": [
    { "id": "genesis", "name_fr": "Genese", "name_en": "Genesis", "chapters": 50, "testament": "old" },
    ...
  ]
}
```

### Thematic Verse Index

**Expanded categories (10-15):**
healing, encouragement, prosperity, faith, protection, love, wisdom, praise, peace, family, work, forgiveness, strength, joy

**Index file:**
```
src/data/verse-tags.json
{
  "healing": [
    { "ref": "Esaie 53:5", "book": "isaiah", "chapter": 53, "verse": 5 },
    { "ref": "Psaume 103:3", "book": "psalms", "chapter": 103, "verse": 3 },
    ...

  ...

```

- Each category has 50-200+ tagged verses from across the entire Bible
- Alarm picks verses in rotation from the category pool (no repeats for months)
- Replaces the old hardcoded BUILTIN_PACKS system

### Bible Browser UI

New screen accessible from Packs tab:
- Book list grouped by Old/New Testament
- Tap book -> chapter list
- Tap chapter -> verse list with "+" button per verse to add to custom pack
- Search bar at top (search by keyword or reference like "Jean 3:16")
- Translation dropdown selector at top

### Integration with Alarms

- Alarm type `packId` field evolves: can be a category key ("healing") or a custom pack ID
- When category: verses pulled from verse-tags.json index, resolved against active Bible translation
- When custom pack: verses from user's manually curated list (as today)

## Feature 3: Alarm Display Templates

### Current State
Single alarm-triggered screen: plain background, verse centered, two buttons.

### Design
4 templates the user chooses per alarm (in alarm edit form, new "STYLE" section):

#### Template A: Standard
- Clean background (matches app theme)
- Verse centered, large italic text
- Gold reference
- Two buttons: "Silence" + "Fermer"
- Simple and distraction-free

#### Template B: Appel Entrant (Incoming Call)
- Mimics iOS incoming call screen
- Top: blurred background + circular icon with category symbol
- Middle: alarm name as "caller", category as subtitle, verse text below
- Bottom: two round buttons
  - Green "Prier" (pray/accept) -> enters prayer mode (timer + verse stays)
  - Red "Plus tard" (decline) -> dismisses alarm
  - Small text button "Rappeler dans 5 min" (snooze) -> reschedules +5min
- Subtle pulse animation on the icon

#### Template C: Reveil (Wake-up)
- Style: iPhone Clock alarm
- Large time display at top
- Verse below in medium text
- Two buttons: "Snooze (+5 min)" + "Arreter"
- Minimal, functional

#### Template D: Immersif (Immersive)
- Animated gradient mesh background (gold/amber tones, slow pulse)
- Floating light particles effect (subtle)
- Verse in large text with soft glow behind
- Slide-to-pray gesture at bottom (horizontal slider)
- After sliding: calm mode with prayer timer
- Most visually striking template

### Alarm Type Update
```typescript
export interface Alarm {
  // ... existing fields
  template: 'standard' | 'incoming-call' | 'wake-up' | 'immersive';
}
```

### New Section in Alarm Form
- "STYLE" section with 4 preview thumbnails (small cards showing template preview)
- Tap to select, gold border on selected

## Feature 4: Internationalisation (i18n) FR/EN

### Design

**Translation files:**
```
src/i18n/
  fr.json   # { "home.title": "Prayer Room", "alarm.create": "Creer l'alarme", ... }
  en.json   # { "home.title": "Prayer Room", "alarm.create": "Create alarm", ... }
  index.ts  # t() helper, locale detection, language context
```

**Implementation:**
- `LanguageContext` wrapping the app (like ThemeContext)
- `t(key)` function that reads from active locale file
- `useLanguage()` hook returning `{ language, t, setLanguage }`
- First launch: detect via `expo-localization` Localization.locale
- Persist choice in AsyncStorage (Settings.language field)
- Manual override in Settings screen: "Langue / Language" selector

**Settings type update:**
```typescript
export interface Settings {
  theme: 'dark' | 'light';
  language: 'fr' | 'en';
  lastVerseIndex: Record<string, number>;
}
```

**Impact:**
- All hardcoded French strings in screens/components replaced with `t()` calls
- Bible default translation follows app language (FR -> ls1910, EN -> kjv)
- Category names translated via i18n keys
- Alarm template button labels translated

## Feature 5: Native Alarm Sounds

### Current State
NATIVE_SOUNDS list has 5 entries (Son par defaut, Tri-tone, Cloche, Carillon, Harpe) but they're just names — no actual audio files. BUILTIN_TRACKS reference 5 MP3s that don't exist in assets/audio/.

### Design

**Bundled sounds (real audio files needed):**

Notification/alert sounds (short, 3-5 seconds):
- `default.mp3` - Soft chime
- `tri-tone.mp3` - Three-tone alert
- `bell.mp3` - Church bell
- `chime.mp3` - Wind chime
- `harp.mp3` - Harp arpeggio
- `shofar.mp3` - Shofar horn (biblical)
- `gentle-ping.mp3` - Soft ping

Worship/ambient tracks (longer, 30-60 seconds, for prayer mode):
- `worship-piano-1.mp3` - Soft piano worship
- `worship-piano-2.mp3` - Gentle piano melody
- `gentle-worship.mp3` - Acoustic worship
- `prayer-ambient.mp3` - Ambient prayer atmosphere
- `meditation-calm.mp3` - Calm meditation

**Source options:**
- Generate via AI music tools (royalty-free)
- Source from freesound.org or similar CC0 libraries
- Commission custom sounds

**Audio plays differently per template:**
- Standard/Wake-up: notification sound plays once or loops until dismissed
- Incoming Call: ringtone-style loop (like phone ring)
- Immersive: ambient track plays, fades when entering prayer mode

## Priority Order

1. **Alarm reliability** (core mission) - Ensure notifications fire reliably at every interval
2. **i18n FR/EN** - Foundation that affects all other features
3. **Custom interval picker** - Small, self-contained
4. **Alarm templates** - Visual impact, user-facing
5. **Bible + thematic index** - Largest feature, most files
6. **Native sounds** - Requires sourcing actual audio files
